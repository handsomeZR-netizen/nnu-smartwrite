# Netlify 快速部署指南

## 一键部署按钮（可选）

如果你想为项目添加一键部署按钮，可以在 README.md 中添加：

```markdown
[![Deploy to Netlify](https://www.netlify.com/img/deploy/button.svg)](https://app.netlify.com/start/deploy?repository=https://github.com/YOUR_USERNAME/nnu-smartwrite)
```

## 环境变量配置

### 在 Netlify Dashboard 设置

1. 进入你的站点
2. 点击 **Site configuration** → **Environment variables**
3. 点击 **Add a variable**
4. 添加以下变量：

```
Key: DEEPSEEK_API_KEY
Value: sk-your-actual-api-key-here
Scopes: All scopes (或选择 Production)
```

```
Key: NEXT_PUBLIC_APP_URL
Value: https://your-site.netlify.app
Scopes: All scopes
```

### 使用 Netlify CLI 设置（可选）

```bash
# 安装 Netlify CLI
bun install -g netlify-cli

# 登录
netlify login

# 链接到站点
netlify link

# 设置环境变量
netlify env:set DEEPSEEK_API_KEY "sk-your-key-here"
netlify env:set NEXT_PUBLIC_APP_URL "https://your-site.netlify.app"
```

## 构建配置

### 自动检测

Netlify 会自动读取 `netlify.toml` 文件中的配置。

### 手动配置（如果需要）

如果自动检测失败，可以在 Netlify Dashboard 手动设置：

**Site configuration** → **Build & deploy** → **Build settings**

- **Base directory**: `nnu-smartwrite`
- **Build command**: `bun install --frozen-lockfile && bun run build`
- **Publish directory**: `nnu-smartwrite/.next`
- **Functions directory**: `nnu-smartwrite/.netlify/functions`

## 部署分支策略

### 生产部署

- **分支**: `main` 或 `master`
- **自动部署**: 推送到主分支时自动触发
- **URL**: 主域名（如 `https://nnu-smartwrite.netlify.app`）

### 预览部署

- **分支**: 任何非主分支（如 `dev`, `feature/*`）
- **自动部署**: 推送时自动创建预览
- **URL**: 唯一的预览 URL（如 `https://deploy-preview-123--nnu-smartwrite.netlify.app`）

### Pull Request 部署

- 每个 PR 自动创建预览部署
- 在 PR 中显示部署状态和预览链接
- 合并后自动清理

## 常用命令

### 本地测试生产构建

```bash
# 构建
bun run build

# 启动生产服务器
bun start
```

### 使用 Netlify CLI 本地测试

```bash
# 安装 Netlify CLI
bun install -g netlify-cli

# 本地运行（模拟 Netlify 环境）
netlify dev

# 本地测试构建
netlify build

# 部署到 Netlify
netlify deploy

# 部署到生产环境
netlify deploy --prod
```

## 部署状态检查

### 在 Netlify Dashboard

1. 进入 **Deploys** 页面
2. 查看最新部署状态
3. 点击部署查看详细日志

### 部署状态标识

- 🟢 **Published**: 部署成功并已发布
- 🟡 **Building**: 正在构建中
- 🔴 **Failed**: 构建失败
- ⚪ **Queued**: 等待构建

## 查看日志

### 构建日志

**Deploys** → 选择部署 → **Deploy log**

### 函数日志

**Functions** → 选择函数 → **Function log**

### 实时日志（使用 CLI）

```bash
netlify logs:function evaluate
```

## 回滚部署

如果新部署有问题：

1. 进入 **Deploys** 页面
2. 找到之前的稳定部署
3. 点击 **Publish deploy** 按钮
4. 确认回滚

## 自定义域名

### 添加域名

1. **Site configuration** → **Domain management**
2. 点击 **Add domain**
3. 输入域名（如 `smartwrite.nnu.edu.cn`）
4. 按照提示配置 DNS

### DNS 配置

**A 记录**（根域名）:
```
Type: A
Name: @
Value: 75.2.60.5
```

**CNAME 记录**（子域名）:
```
Type: CNAME
Name: smartwrite
Value: your-site.netlify.app
```

### SSL 证书

Netlify 自动为所有域名提供免费的 Let's Encrypt SSL 证书。

## 故障排查

### 构建失败

1. 检查构建日志中的错误信息
2. 确认 Node 版本正确（20）
3. 验证所有依赖已安装
4. 本地运行 `bun run build` 测试

### 环境变量问题

1. 确认变量名拼写正确
2. 客户端变量必须以 `NEXT_PUBLIC_` 开头
3. 更新后需要重新部署
4. 检查变量作用域（Production/Deploy Preview/Branch deploys）

### 函数超时

默认超时：10 秒（免费套餐）

如果 API 调用超时：
1. 优化 API 调用逻辑
2. 考虑升级到 Pro 套餐（26 秒超时）
3. 实现重试机制

### 404 错误

1. 检查 `netlify.toml` 重定向规则
2. 确认 SPA fallback 配置正确
3. 验证路由配置

## 成本管理

### 免费套餐限制

- 100 GB 带宽/月
- 300 分钟构建时间/月
- 125,000 次函数调用/月

### 监控使用量

**Site configuration** → **Usage**

### 优化建议

- 启用缓存减少函数调用
- 优化图片减少带宽
- 使用 CDN 加速静态资源

## 安全最佳实践

1. ✅ 定期轮换 API 密钥
2. ✅ 启用 HTTPS（自动）
3. ✅ 配置安全头（已在 `netlify.toml` 中）
4. ✅ 限制函数调用频率
5. ✅ 监控异常流量

## 备份策略

2. Netlify 保留所有历史部署
3. 定期导出环境变量配置
4. 保存 `netlify.toml` 配置文件
